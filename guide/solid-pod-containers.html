# Solid Pod Containers

All containers are in My Shop folder.

# contacts.json

Holds all contacts with their access to the current user's shop/shops and the current user's access to given contact's shop/shops. It is updated only when either the current user searches for a shop or someone request an access to the current user's shop (RequestingAccess.vue pops up).

Example:

{
    "https://bob.solidcommunity.net/profile/card#me": {
        "myShops": [{ 
            "shop": "shop1",
            "status": "accepted"
        }],
        "theirShops": [{
            "shop": "shop2",
            "status": "rejected"
        }, {
            "shop": "shop3",
            "status": "pending"
        }]
    }
}

Here, the example shows that the current user has accepted a https://bob.solidcommunity.net/profile/card#me's request and https://bob.solidcommunity.net/profile/card#me has a read access to shop1. The current user has sent a request to https://bob.solidcommunity.net/profile/card#me but https://bob.solidcommunity.net/profile/card#me rejected his request for shop2 and the request for shop3 is pending which means he does not decided yet.

# Mnemonic

Mnemonic is a Solid Dataset and holds the mnemonic phrase (generated from createWallet function) inside the mnemonic Solid Thing.

Example:

{
    "phrase": "text1 text2 text3 text4 text5 text6 text7 text8 text9 text10 text11 text12",
    "path": "m/44'/60'/0'/0/0",
    "locale": "en",
    "index": "1"
}

# My Info

My Info is a Solid Dataset and holds the shipping information (saved by setMyInfo function) inside the myinfo Solid Thing.

Example:

{
    "firstName": "John",
    "lastName": "Doe",
    "phoneNumber": "+10123456789",
    "address": "1600 Pennsylvania Avenue NW, Washington, D.C."
}

# My Orders

Solid Dataset which has an another Solid Dataset with the name of a Solid WebId. The inner folder has a folder with the name of a shop owned by Solid WebId which holds information about the purchased product in this format:

Example

{
    "26e1bdf537f22fc9e36af90a7adf914332649e17": {
        "shopname": "books",
        "name": "book1"
    }
}

The json file is saved only when a product is purchased.

# Product Images

All products images. Example folder structure:

Product Images -> books -> book1 -> image.jpg

Second Solid Dataset is the shop's name and the third is the product's name.

# Shop Covers

All shop's images. Example folder structure:

Shop Covers -> books -> image.jpg

Second Solid Dataset is the shop's name.

# Shops Details

Holds the public key and an index which is the number used when generating the wallet by mnemonic phrase.

Example folder structure:

Shops Details -> books -> address

Address is a Solid Thing which holds the actual information.

Example:

{
    "address": "0x70660481C482338fE4c289b4e4838FDb2085C63e",
    "index": 1
}

# Shops

A Solid Dataset which has data about the details of the products.

Example folder structure:

Shops -> books -> book1

The second folder is a Solid Dataset which is the shop's name and the third folder is a Solid Thing which is the product's name.

book1 holds the actual information.

Example text structure:

{
    "name": "book1",
    "description": "desc",
    "currency": "MATIC",
    "price": "0150000000000000000",
    "fromDate": "2022-08-03T21:00:00.000Z",
    "toDate": "2022-08-04T21:00:00.000Z",
    "count": 3,
    "shipping": "Myshop shipping",
    "hash": "26e1bdf537f22fc9e36af90a7adf914332649e17"
}
